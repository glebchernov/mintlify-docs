# Salsa as a Service API v2.0

The v2.0 API adds GraphQL support, enhanced webhooks, and new AI forecasting endpoints.

## Base URL

```
https://api.salsaasaservice.com/v2
```

<Info>
**New in v2.0**: GraphQL endpoint at `https://api.salsaasaservice.com/graphql`
</Info>

## Authentication

All API requests require authentication using your API key.

<Note>
**API Key**: Get your API key from the [Developer Settings](https://app.salsaasaservice.com/settings/api) in your dashboard.
</Note>

### Authentication Header

Include your API key in the Authorization header:

<CodeGroup>
```bash cURL
curl -X GET https://api.salsaasaservice.com/v2/ingredients \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

headers = {"Authorization": "Bearer YOUR_API_KEY"}
response = requests.get(
    "https://api.salsaasaservice.com/v2/ingredients",
    headers=headers
)
```

```javascript JavaScript
const response = await fetch(
    "https://api.salsaasaservice.com/v2/ingredients",
    { headers: { "Authorization": "Bearer YOUR_API_KEY" } }
);
```
</CodeGroup>

## Rate Limits

- **Free Plan**: 1,000 requests per hour
- **Professional Plan**: 10,000 requests per hour
- **Enterprise Plan**: 100,000 requests per hour

Rate limit headers are included in all responses:

```http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

## Inventory Management

### Get All Ingredients

Retrieve a list of all ingredients in your inventory.

<CodeGroup>
```bash cURL
curl -X GET https://api.salsaasaservice.com/v2/ingredients \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

response = requests.get(
    "https://api.salsaasaservice.com/v2/ingredients",
    headers={"Authorization": "Bearer YOUR_API_KEY"}
)
ingredients = response.json()
```

```javascript JavaScript
const response = await fetch(
    "https://api.salsaasaservice.com/v2/ingredients",
    { headers: { "Authorization": "Bearer YOUR_API_KEY" } }
);
const ingredients = await response.json();
```
</CodeGroup>

**Response:**

```json
{
  "data": [
    {
      "id": "ing_123",

      "quality_score": 8.5
    }
  ],

    "page": 1,

```

### Update Ingredient Stock

Update the stock level for a specific ingredient.

<CodeGroup>
```bash cURL
curl -X PUT https://api.salsaasaservice.com/v2/ingredients/ing_123/stock \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"quantity": 30.0, "unit": "lbs", "notes": "New shipment received"}'
```

```python Python
response = requests.put(
    "https://api.salsaasaservice.com/v2/ingredients/ing_123/stock",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={"quantity": 30.0, "unit": "lbs", "notes": "New shipment received"}
)
```

```javascript JavaScript
const response = await fetch(
    "https://api.salsaasaservice.com/v2/ingredients/ing_123/stock",
    {
      method: "PUT",
      headers: {
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
      },
      body: JSON.stringify({ quantity: 30.0, unit: "lbs", notes: "New shipment received" })
    }
);
```
</CodeGroup>

**Response:**

```json
{
  "id": "ing_123",
  "name": "Roma Tomatoes",
  "current_stock": 30.0,
  "unit": "lbs",
  "updated_at": "2024-01-15T11:00:00Z"
}
```

### Complete Production Batch

```http
PUT /production/batches/{batch_id}/complete
```

**Request Body:**

```json
{
  "actual_quantity": 4.8,
  "quality_score": 9.2,
  "notes": "Excellent batch, perfect consistency",
  "completed_at": "2024-01-15T09:30:00Z"
```

## Error Handling

The API uses standard HTTP status codes and returns detailed error information.

### Error Response Format

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing required fields",
    "details": {
      "missing_fields": ["name", "category"]
    }
  }
}
```

### Common Error Codes

<Tabs>
  <Tab title="Client Errors (4xx)">
    - `400 Bad Request` — Invalid request data
    - `401 Unauthorized` — Missing or invalid API key
    - `403 Forbidden` — Insufficient permissions
    - `404 Not Found` — Resource not found
    - `422 Unprocessable Entity` — Validation errors
    - `429 Too Many Requests` — Rate limit exceeded
  </Tab>
  <Tab title="Server Errors (5xx)">
    - `500 Internal Server Error` — Server error
    - `502 Bad Gateway` — Upstream service error
    - `503 Service Unavailable` — Service temporarily unavailable
  </Tab>
</Tabs>

## Webhooks

Configure webhooks to receive real-time notifications about important events.

### Supported Events

- `ingredient.low_stock` — When ingredient stock falls below threshold
- `production.batch_completed` — When a production batch is completed
- `quality.alert` — When quality scores fall below acceptable levels
- `inventory.expiry_warning` — When ingredients are approaching expiry

### Webhook Configuration

```json
{
  "url": "https://your-app.com/webhooks/salsa",
  "events": ["ingredient.low_stock", "production.batch_completed"],
  "secret": "your_webhook_secret"
}
```
