# Troubleshooting Guide - v2.0

This guide helps you resolve common issues with Salsa as a Service v2.0, including AI forecasting and IoT sensor problems. If you can't find a solution here, our support team is ready to help.

## Quick Diagnostics

Before diving into specific issues, run these quick checks:

<Steps>
  <Step title="Check System Status">
    Visit our [status page](https://status.salsaasaservice.com) to see if there are any ongoing service issues.
  </Step>
  <Step title="Verify Internet Connection">
    Ensure you have a stable internet connection and can access other websites.
  </Step>
  <Step title="Clear Browser Cache">
    Clear your browser cache and cookies, then try logging in again.
  </Step>
  <Step title="Check Browser Compatibility">
    Ensure you're using a supported browser (Chrome, Firefox, Safari, Edge).
  </Step>
</Steps>

## Login and Authentication

<AccordionGroup>
  <Accordion title="Can't Log In">
    **Problem**: Unable to log into your account

    **Solutions:**

    1. **Verify Credentials** — Double-check your email address and password. Ensure Caps Lock is off.
    2. **Reset Password** — Use the "Forgot Password" link on the login page to reset your password.
    3. **Check Email** — Look for account verification emails in your spam folder.
    4. **Contact Support** — If the issue persists, contact support with your account email address.

    <Warning>
    **Security Note**: Never share your password with anyone. Our support team will never ask for your password.
    </Warning>
  </Accordion>
  <Accordion title="Two-Factor Authentication Issues">
    **Problem**: Issues with 2FA codes or setup

    1. **Code Not Working**: Ensure your device's time is synchronized
    2. **Lost Device**: Use backup codes or contact support
    3. **App Issues**: Try a different authenticator app (Google Authenticator, Authy)
    4. **Setup Problems**: Follow our [2FA setup guide](/guides/two-factor-auth)

    <Tip>If you're using SSO, make sure your IT admin has provisioned your account in the identity provider.</Tip>
  </Accordion>
  <Accordion title="Account Locked">
    **Problem**: Account locked after too many failed login attempts

    1. Wait 30 minutes for the automatic unlock
    2. Use the "Forgot Password" flow to reset credentials
    3. If using SSO, contact your IT administrator
    4. For urgent access, call our support line

    <Warning>After 10 failed attempts, your account is locked for 24 hours. Contact support if you need immediate access.</Warning>
  </Accordion>
</AccordionGroup>

## Data Synchronization

<AccordionGroup>
  <Accordion title="POS Integration Issues">
    **Problem**: POS data not syncing with Salsa as a Service

    <Tabs>
      <Tab title="Square">
        1. **Check Square Connection** — Verify the Square integration is still connected in your dashboard.
        2. **Reauthorize Integration** — Disconnect and reconnect your Square account.
        3. **Check API Permissions** — Ensure Square API permissions include inventory management.
      </Tab>
      <Tab title="Toast">
        1. **Verify Toast API Key** — Check that your Toast API credentials are valid and not expired.
        2. **Check Menu Mapping** — Ensure salsa menu items are properly mapped to inventory items.
      </Tab>
      <Tab title="Clover">
        1. **Update Clover App** — Ensure the Salsa as a Service app is updated in your Clover dashboard.
        2. **Check Permissions** — Verify the app has necessary permissions for data access.
      </Tab>
      <Tab title="General">
        1. **Check Network Connection** — Ensure stable internet connection for both systems.
        2. **Review Sync Logs** — Check the integration logs for error messages.
        3. **Test Manual Sync** — Try manually triggering a data sync from your dashboard.
      </Tab>
    </Tabs>
  </Accordion>
  <Accordion title="Inventory Sync Problems">
    **Problem**: Inventory levels not updating correctly

    **Common Causes:**
    - Network connectivity issues
    - API rate limiting
    - Incorrect unit conversions
    - Data validation errors

    **Solutions:**

    1. **Check Sync Status** — Navigate to **Settings > Integrations** and check the sync status.
    2. **Manual Sync** — Try manually syncing inventory data from the integrations page.
    3. **Review Error Logs** — Check the error logs for specific sync failures.
    4. **Contact Support** — If issues persist, contact support with your sync logs.
  </Accordion>
</AccordionGroup>

## AI Forecasting Issues

<Note>These issues are specific to v2.0 AI features.</Note>

<AccordionGroup>
  <Accordion title="Inaccurate Forecasts">
    **Problem**: AI forecasts don't seem accurate

    1. **Check Historical Data** — Ensure you have at least 90 days of historical data imported.
    2. **Verify Data Sources** — Confirm all data sources (POS, reservations, etc.) are connected and syncing.
    3. **Learning Period** — The AI requires 2-4 weeks to train initially. Accuracy improves over time.
    4. **External Factors** — Check if unusual events (holidays, weather) are affecting predictions.

    <Tip>The AI achieves 70-80% accuracy with 90 days of data and improves to 95%+ after 6 months of learning.</Tip>
  </Accordion>
  <Accordion title="Forecast Not Generating">
    **Problem**: No forecast data appearing in the dashboard

    1. **Check Feature Toggle** — Ensure demand forecasting is enabled in **AI Features > Demand Forecasting**.
    2. **Data Requirements** — You need at least 30 days of historical data for the first forecast.
    3. **Processing Time** — The first forecast takes up to 24 hours to generate.
    4. **Contact AI Support** — For persistent issues: ai-support@salsaasaservice.com
  </Accordion>
</AccordionGroup>

## IoT Sensor Issues

<Note>These issues are specific to v2.0 IoT features.</Note>

<AccordionGroup>
  <Accordion title="Sensor Offline">
    **Problem**: IoT sensor not sending data

    <Tabs>
      <Tab title="Basic Checks">
        1. Check the sensor battery level
        2. Ensure the sensor is within network range
        3. Restart the sensor
        4. Verify WiFi/Bluetooth network status
      </Tab>
      <Tab title="Reconnection">
        1. Open the mobile app
        2. Go to **Settings > IoT Sensors**
        3. Find the offline sensor
        4. Tap "Reconnect"
        5. Follow the pairing instructions
      </Tab>
      <Tab title="Recalibration">
        If the sensor is online but readings are incorrect:
        1. Access sensor settings
        2. Select "Recalibrate"
        3. Follow the guided calibration procedure
        4. Test with a reference reading
      </Tab>
    </Tabs>
  </Accordion>
  <Accordion title="Incorrect Sensor Readings">
    **Problem**: Sensor sends data but values seem wrong

    1. **Recalibrate the sensor** — Run a calibration routine from the sensor settings
    2. **Check placement** — Ensure the sensor is correctly positioned
    3. **Check for interference** — Move away from heat sources or humidity
    4. **Replace battery** — A low battery can cause inaccurate readings

    <Warning>If a temperature sensor shows readings outside safe food handling ranges, pause production and verify manually before continuing.</Warning>
  </Accordion>
</AccordionGroup>

## Production Issues

<AccordionGroup>
  <Accordion title="Quality Control Problems">
    **Problem**: Quality scores not recording correctly

    1. **Check Device Calibration** — Ensure quality testing devices are properly calibrated
    2. **Review Testing Procedures** — Verify staff are following correct testing protocols
    3. **Update Software** — Ensure the mobile app is updated to the latest version
    4. **Check Permissions** — Verify staff have proper permissions for quality testing
  </Accordion>
  <Accordion title="Batch Tracking Issues">
    **Problem**: Production batches not tracking properly

    <Note>Each batch must have a unique batch number and valid recipe assignment.</Note>

    1. **Verify Batch Data** — Check that all required batch information is entered correctly.
    2. **Check Recipe Assignment** — Ensure the batch is assigned to a valid recipe.
    3. **Review Staff Permissions** — Verify the assigned chef has proper permissions for batch management.
  </Accordion>
</AccordionGroup>

## Mobile App Issues

<AccordionGroup>
  <Accordion title="App Crashes">
    **Problem**: Mobile app crashes or freezes

    1. **Update App** — Ensure you're running the latest version of the Salsa as a Service mobile app.
    2. **Restart App** — Force close the app and restart it.
    3. **Clear App Data** — Clear app cache and data (you'll need to log in again).
    4. **Reinstall App** — Uninstall and reinstall the app from your device's app store.
  </Accordion>
  <Accordion title="Offline Mode Issues">
    **Problem**: App not working in offline mode

    1. **Check Offline Data** — Ensure critical data is downloaded for offline use
    2. **Sync Before Going Offline** — Manually sync data before losing internet connection
    3. **Check Storage Space** — Ensure sufficient device storage for offline data
    4. **Update App** — Use the latest app version with improved offline capabilities
  </Accordion>
</AccordionGroup>

## Performance Issues

<AccordionGroup>
  <Accordion title="Slow Loading Times">
    **Problem**: Dashboard or app loading slowly

    <Tabs>
      <Tab title="Browser Fixes">
        1. Clear cookies, cache, and browsing data
        2. Temporarily disable browser extensions to identify conflicts
        3. Test with a different browser to isolate the issue

      <Tab title="Network Fixes">

        2. Try a different network (mobile hotspot, etc.)

      <Tab title="Data Fixes">

        2. Narrow date ranges on reports
        3. Archive old production data
        4. Contact support for database optimization
      </Tab>
    </Tabs>
  </Accordion>
</AccordionGroup>

## API Integration Issues

<AccordionGroup>
  <Accordion title="Authentication Errors (401)">
    **Problem**: API requests returning 401 Unauthorized

    1. **Check API Key** — Verify the API key is correct and not expired
    2. **Check Permissions** — Ensure the API key has necessary permissions
    3. **Check Rate Limits** — Verify you haven't exceeded rate limits
    4. **Check Request Format** — Ensure requests are properly formatted
  </Accordion>
  <Accordion title="Data Format Errors">
    **Problem**: API requests failing due to data format errors

    **Common issues:** incorrect date formats, invalid data types, missing required fields, invalid JSON syntax.

    **Example of correct format:**

    ```javascript
    const requestData = {
      name: "House Salsa",
      category: "salsa",
      batch_size: 5.0,  // Use numbers, not strings
      production_date: "2024-01-15T08:00:00Z",  // ISO 8601 format
      ingredients: [
        {
          ingredient_id: "ing_123",
          quantity: 2.0,
          unit: "lbs"
        }
      ]
    };
    ```
  </Accordion>
  <Accordion title="GraphQL Query Errors">
    **Problem**: GraphQL queries failing or returning errors

    1. **Authentication** — Verify the Bearer token in your headers
    2. **Invalid Fields** — Use introspection to check available fields
    3. **Depth Limit** — Queries are limited to 10 levels of depth
    4. **Rate Limits** — Respect GraphQL-specific rate limits

    <Tip>Use the interactive GraphQL explorer at `https://api.salsaasaservice.com/graphql/explorer` to test queries before implementing them.</Tip>
  </Accordion>
</AccordionGroup>

When contacting support, please include:

1. **Account Information**: Your account email and restaurant name
2. **Issue Description**: Detailed description of the problem
3. **Steps Taken**: What troubleshooting steps you've already tried
4. **Error Messages**: Any error messages or codes you're seeing
5. **Screenshots**: Screenshots of the issue (if applicable)
6. **Browser/Device Info**: Browser version and device information

**Support Channels:**

- 📧 **Email**: support@salsaasaservice.com
